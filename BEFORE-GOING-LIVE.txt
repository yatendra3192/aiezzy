YOU NEED TO DO BEFORE LAUNCH
-----------------------------

1. GENERATE A STRONG NEXTAUTH_SECRET
   Your current secret literally says "change-in-production".
   Run this command to generate a proper one:
     openssl rand -base64 48
   Then set it as NEXTAUTH_SECRET in Railway environment variables.
   DO NOT reuse the dev secret.

2. SET NEXTAUTH_URL IN RAILWAY
   Set NEXTAUTH_URL=https://aiezzy.com in Railway env vars.
   This is required — password reset and email verification links
   will break without it (we removed the localhost fallback).

3. ROTATE ALL API KEYS
   Your .env.local has been on disk with real keys. Even though
   .gitignore excludes it, regenerate these keys for production:
     - Google Maps API key (restrict to your domain in Google Cloud Console)
     - Supabase anon key (if compromised)
     - OpenAI API key
     - Amadeus API key + secret
     - FLIGHTS_API_KEY
     - Google OAuth client secret
   Set all of them as Railway env vars, NOT in .env.local.

4. RESTRICT GOOGLE MAPS API KEY
   Go to Google Cloud Console > Credentials > Your API key:
     - Add HTTP referrer restriction: aiezzy.com/*
     - Restrict to only the APIs you use (Places, Directions, Geocoding, Maps JS)
   Without this, anyone who finds your key can burn your quota.

5. ADD RATE LIMITING
   Currently zero rate limiting. Someone can spam your AI endpoints
   and burn your OpenAI credits. Options:
     a) Upstash Redis (free tier, works with Railway)
        npm install @upstash/ratelimit @upstash/redis
     b) Vercel KV if you move to Vercel
   Priority routes to rate limit:
     - /api/ai/* (most expensive — OpenAI calls)
     - /api/flights (Amadeus quota)
     - /api/nearby (Google Hotels scraper)
     - /api/auth/signup (prevent spam signups)
     - /api/auth/forgot-password (prevent email spam)

6. GENERATE PWA ICONS
   manifest.json references icon-192.png and icon-512.png but
   they don't exist. PWA install will fail on mobile.
   Generate from your logo and place in /public/:
     - /public/icon-192.png (192x192 px)
     - /public/icon-512.png (512x512 px)

7. SET UP SUPABASE RLS PROPERLY
   Double check that Row Level Security is enabled on ALL tables:
     - profiles
     - trips
     - trip_destinations
     - trip_transport_legs
   Make sure policies only allow users to access their own data.

8. SET UP CUSTOM DOMAIN + SSL
   In Railway:
     - Add custom domain aiezzy.com
     - Verify DNS records
     - SSL is automatic on Railway
   Update NEXTAUTH_URL to match.

9. SET UP ERROR MONITORING
   You have Sentry support (NEXT_PUBLIC_SENTRY_DSN env var).
   Create a Sentry project and add the DSN to Railway env vars.
   This will catch errors in production that you won't see otherwise.

10. SET UP GOOGLE ANALYTICS
    You have GA support (NEXT_PUBLIC_GA_ID env var).
    Create a GA4 property and add the measurement ID to Railway.
